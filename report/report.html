1.　基本的なHTMLとCSSの理解

  • HTMLとCSSはウェブデザインにおいて基本的な技術であり、これらの言語を理解することで自身のアイデアを具現化することが可能となります。私はこれらの言語を学び、ポートフォリオ作成に活かすことで、自分のスキルや経験をより効果的に表現できるようになりました。

2.　ポートフォリオ作成時の成功談

  • ポートフォリオを構築する過程で、ページ間の行き来やページ内の紐づけに特に注意を払いました。各プロジェクトやスキルセットへのアクセスが直感的で迅速できるように、ナビゲーションバーや専用のセクションリンクを導入しました。これにより、訪問者は興味を持った領域に瞬時にアクセスでき、閲覧のしやすさが向上しました。 特に、EamyCode部分では、ホームタグとの連動を工夫しました。ホームタグからのクリックによって、Home画面に瞬時にいどうできるようになりました。。 また、X（旧Twitter）は適切にURLを添付しており、訪問者が追加情報を得ることが容易になっています。これにより、ポートフォリオは自己紹介やプロジェクトのハイライトだけでなく、他のプラットフォームや資料とも連携した総合的な情報源としての役割を果たしています。 以上の取り組みによって、ポートフォリオは私自身のブランディングだけでなく、訪れる人にとって使いやすく、興味を引く魅力的なウェブ環境となりました。

3.　失敗談と失敗からの経験

  • 初めてのポートフォリオ作成では、情報の過不足やデザインの統一性の欠如など、様々な課題に直面しました。その中でも、特にヘッダーにおいてCSSのhover機能を活用し、マウスオーバー時に倍率が変化するエフェクトを実装しようとしましたが、うまくいきませんでした。この課題に対処するために、MDN Web Docsや関連する書籍を参照しながら、正しい構文とプロパティの使用方法を学び、結果的にヘッダータグにhover機能を導入しました。この経験から、エラーに直面した際には適切なリソースを活用する重要性を理解し、積極的に知識を補完する姿勢がポートフォリオの品質向上に繋がったと考えています。

4.　HTMLとCSSを学んだうえで今後やってみたいこと

  • HTMLとCSSの基礎を身につけたら、JavaScriptやフレームワークの学習にも挑戦して、より動的でインタラクティブなウェブサイトを構築したいと考えています。これにより、より洗練されたユーザーエクスペリエンスを提供できるようになります。
